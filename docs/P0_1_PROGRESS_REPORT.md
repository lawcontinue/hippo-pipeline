### 2. 实施复杂度分析

**技术难点**:
1. **KV cache 批量支持**: 当前的 KV cache 实现假设 batch=1
   - **解决方案**: 先实现简化版（每次重建 cache），验证效果后再优化

2. **Speculative execution 兼容**: 当前使用了 spec_task 预计算
   - **解决方案**: 先实现无 speculative execution 的版本，验证批量处理效果

3. **lm_head 批量压缩**: 需要循环处理每个 token 的 top-k
   - **解决方案**: 使用循环（简洁但可能慢），或者优化为一次性处理

**估计时间**: 2-3 小时
